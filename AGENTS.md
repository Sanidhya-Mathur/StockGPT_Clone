# AGENTS: How to work with StockGPT_Clone

Purpose: quick, actionable guidance for AI coding agents to be immediately productive in this repository.

Checklist for an agent
- Identify the intended run-target (CLI vs API) before changing code.
- Use `config/default.yaml` as the canonical config for examples and tests.
- Keep artifacts in `artifacts/` (models, `scaler.joblib`, `metadata.json`).

Big picture (how this repo is organized)
- CLI entrypoint: `cli.py` — dispatches `train`, `evaluate`, `predict` and calls functions in `stockgpt_clone.*`.
- Library core: `stockgpt_clone/` — contains `data.py`, `train.py`, `predict.py`, `evaluate.py`, and `models/`.
- API: `api/app.py` — a FastAPI app exposing `/predict` and `/health` that calls the same library functions.
- Artifacts and metadata: `artifacts/` (models saved by Keras or joblib + `metadata.json`).

Key run & debug commands (use PowerShell on developer machines)
- Create venv & install deps:
  python -m venv .venv; .\.venv\Scripts\Activate.ps1; pip install -r requirements.txt
- Run CLI (examples):
  python cli.py train --config config/default.yaml
  python cli.py predict --config config/default.yaml
- Run API server locally:
  uvicorn api.app:app --reload --port 8000
- Inspect artifacts:
  python scripts\inspect_artifacts.py artifacts

Important code patterns & examples
- Config-driven flows: top-level commands call functions like
  `stockgpt_clone.train.train_from_config(config_path)` (see `cli.py`).
- Prediction sequence (LSTM):
  load scaler: `scaler = joblib.load(scaler_path)`
  load model: `model = load_lstm_model(model_path)`
  transform window → `model.predict(...)` → inverse transform
  (see `stockgpt_clone/predict.py`).
- Model persistence:
  - LSTM uses Keras `model.save(path)`; loader tries common extensions (`.keras`) (see `models/lstm_model.py`).
  - XGBoost uses `joblib.dump` / `joblib.load` (see `models/xgb_model.py`).

Configuration notes (observed keys)
- `config/default.yaml` contains `data`, `model`, `training`, and `artifacts` sections.
- `model.type` toggles code paths: `lstm` vs `xgboost`.

Developer conventions specific to this repo
- Artifacts directory is authoritative — training writes models + `scaler.joblib` + `metadata.json`.
- Reuse library functions in `stockgpt_clone` from both CLI and API; avoid duplicating logic in `api/app.py`.
- Logging uses `stockgpt_clone/utils/logging_utils.py` — configure logger rather than printing.

Integration & external dependencies
- Heavy ML libs: `tensorflow` (LSTM) and `xgboost` (tabular). Persisted with Keras model files and joblib.
- Data ingestion: `yfinance` in `stockgpt_clone/data.py` for fetching ticker history.

Notes for edits and tests
- There are no unit tests or CI configs in repo root; run small end-to-end checks instead: train a tiny model with `config/default.yaml` (reduce epochs) and run `uvicorn` + a `/predict` request.
- Avoid changing artifact naming conventions (extensions matter for loaders).

Files to inspect first when making changes
- `cli.py`, `api/app.py`, `config/default.yaml`, `stockgpt_clone/train.py`, `stockgpt_clone/predict.py`, `stockgpt_clone/models/*`, `scripts/inspect_artifacts.py`.
